# v0.5.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把当前 main 分支代码状态正式发布为 `v0.5.0` 到 Maven Central + GitHub Releases, 同步更新 live docs 版本引用, 下一 dev 版本切到 `0.6.0-SNAPSHOT`.

**Architecture:** A 步: 手动改 5 个 live 文档的版本引用 → B 步: `./scripts/release.sh 0.5.0 --next 0.6.0` 自动改 pom + 跑测试 + commit + tag + 切 SNAPSHOT → C 步: 推 main + tag 触发 CI → D 步: 监控并验证 Maven Central + GitHub Release.

**Tech Stack:** Bash, Maven (`mvn versions:set`, `mvn clean verify` 含 jacoco), Git tag, GitHub Actions (maven-central.yml + release.yml).

参考 spec: `docs/superpowers/specs/2026-07-23-v0.5.0-release-design.md`

## 风险提示

1. **Maven Central 不可撤销**: 推 tag 后 CI 立即发布, 一旦成功发布 `0.5.0` 到 Central, 无法删除/覆盖. 如果发布失败需要新版本号 `0.5.1` 或 `0.6.0`.
2. **jacoco 覆盖率检查**: `mvn clean verify` 会跑覆盖率检查, 如果近期代码改动导致覆盖率下降可能失败. 如果失败, 不要 `--skip-tests`, 而是补测试或排查根因.
3. **macOS sed 语法**: release.sh 用 `sed -i ''` 是 BSD sed 语法(macOS 原生), 在 Linux 上跑会失败. 已确认当前环境是 darwin.
4. **GitHub Secrets 状态**: Task 5 Step 3 必须先验证 4 个 secret 都在, 否则 Task 7 推 tag 后 CI 会失败, 版本号作废.
5. **网络可达性**: CI 跑 GPG 签名需要访问 keyserver, 跑 deploy 需要访问 `central.sonatype.com`. 如果 GitHub Actions runner 网络异常, 发布会失败.
