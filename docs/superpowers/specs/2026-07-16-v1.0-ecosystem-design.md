## 1. 目标

建立完整生态——工具插件化、Skill 编排工作流、成本透明化。三个方向共同构成"可运营"的 Agent 平台。

本次交付范围（三轨并行）：

### Track 1: 成本核算（Cost Accounting）— 主轨
- `CostRecord` 数据模型：每次 LLM 调用的 token 用量 + 费用记录
- `CostStore` SPI：成本记录持久化
- `CostTracker` SPI：记录 + 预算检查
- `CostTrackingLlmClient`：装饰器包装 LlmClient，自动记录每次调用
- `FileCostStore`：JSON 文件存储默认实现
- `BudgetEnforcer`：预算检查（per-user/per-skill/global daily），80%告警/100%拒绝
- `CostSummaryService`：按用户/Skill/时间段聚合
- Controller 端点：`GET /cost/summary` / `GET /cost/users/{id}/summary` / `GET /cost/skills/{name}/summary`
- 配置：`snap-agent.cost.*`

### Track 2: 工作流引擎（Workflow Engine）— 副轨
- `WorkflowDefinition` + `WorkflowStep` 数据模型
- `WorkflowEngine` SPI：执行工作流
- `WorkflowResult` 数据模型
- `YamlWorkflowLoader`：从 YAML 加载工作流定义
- `SimpleWorkflowEngine`：顺序执行 + 条件分支（无循环/审批）
- Controller 端点：`POST /workflows/{name}/run` / `GET /workflows` / `GET /workflows/{name}`
- 配置：`snap-agent.workflows.*`
- 内置工作流：`full-diagnose.yml`

### Track 3: 工具插件元数据（Tool Plugin Metadata）— 轻量
- `ToolPlugin` SPI：插件元数据（name/version/description/toolNames）
- `ToolPluginRegistry`：收集所有 ToolPlugin bean，暴露元数据
- Controller 端点：`GET /tools/plugins`
- 文档：说明 `ToolProvider` + `@Component` 已是自动发现机制

不在本次范围（延后到 v1.0.1+）：
- MCP 协议自有应用插件 — v1.0.1
- 工作流循环/人工审批 — v1.0.1
- 定时/事件触发工作流 — v1.0.1
- 独立插件 JAR（`snap-agent-tool-mysql` 独立模块）— v1.0.2
- 成本实时 SSE 告警推送 — v1.0.1
- DB 成本存储 — v1.0.1

### 6.2 CostTrackingLlmClient (装饰器)

```java
package cn.watsontech.snapagent.boot2x.cost;

/**
 * 装饰 LlmClient, 自动记录每次调用的 token 用量和费用。
 * 不修改 AgentExecutor, 通过 bean 装配替换原始 LlmClient。
 */
public class CostTrackingLlmClient implements LlmClient {
    private final LlmClient delegate;
    private final CostTracker costTracker;
    private final String inputPricePerMillion;  // 每 1M tokens 单价
    private final String outputPricePerMillion;
    private final String cacheReadPricePerMillion;

    @Override
    public LlmResponse chat(LlmRequest request) {
        LlmResponse response = delegate.chat(request);
        // 从 response.getUsage() 提取 token 计数
        // 计算 cost = inputTokens * inputPrice + outputTokens * outputPrice + cacheReadTokens * cachePrice
        // costTracker.record(new CostRecord(...))
        return response;
    }
}
```

**关键**: 装饰器模式不修改 AgentExecutor。autoconfig 中:
- cost.enabled=false → 直接注入原始 LlmClient
- cost.enabled=true → 创建 CostTrackingLlmClient 包装原始 LlmClient, 注入给 AgentExecutor

### 6.3 BudgetEnforcer

```java
public class BudgetEnforcer {
    private final CostStore costStore;
    private final BigDecimal perUserDaily;
    private final BigDecimal perSkillDaily;
    private final BigDecimal globalDaily;

    /** 检查预算, 返回 true 表示在预算内。 */
    public boolean isWithinBudget(String userId, String skillName) {
        long todayStart = startOfTodayMillis();
        // 检查 per-user
        if (perUserDaily != null) {
            BigDecimal userCost = costStore.sumCostByUser(userId, todayStart, System.currentTimeMillis());
            if (userCost.compareTo(perUserDaily) >= 0) return false;
        }
        // 检查 per-skill
        if (perSkillDaily != null) {
            BigDecimal skillCost = costStore.sumCostBySkill(skillName, todayStart, System.currentTimeMillis());
            if (skillCost.compareTo(perSkillDaily) >= 0) return false;
        }
        // 检查 global
        if (globalDaily != null) {
            BigDecimal total = costStore.sumCost(todayStart, System.currentTimeMillis());
            if (total.compareTo(globalDaily) >= 0) return false;
        }
        return true;
    }
}
```

## 7. Starter 层 — 工作流

### 7.1 YamlWorkflowLoader

从 `snap-agent.workflows.dir` (默认 `{upload-skills-dir}/workflows/`) 加载 `.yml` 文件。
解析格式:
```yaml
name: full-diagnose
description: 全链路诊断工作流
steps:
  - name: health-check
    skill: health-check
    inputs:
      service: "${trigger.service}"
  - name: find-root-cause
    skill: error-spike-investigation
    condition: "${health-check.result.contains('error')}"
    inputs:
      timeWindow: "1h"
    onFailure: STOP
  - name: suggest-fix
    skill: code-analysis
    condition: "${find-root-cause.result != null}"
    inputs:
      filePath: "${find-root-cause.result}"
    onFailure: SKIP
```

### 7.2 SimpleWorkflowEngine

顺序执行步骤:
1. 遍历 steps
2. 对每个 step, 先检查 condition (字符串替换 + 简单判断)
3. 解析 inputs 中的 `${trigger.xxx}` 和 `${stepName.result}` 引用
4. 构造 AgentTask, 调用 `agentExecutor.execute()`
5. 存储 `stepName → result` 映射
6. onFailure: STOP → 终止; SKIP → 继续; RETRY → 重试一次
7. 返回 WorkflowResult

**条件解析** (简单, 非 SpEL):
- `${expr != null}` → stepResults.containsKey(stepName) && result != null
- `${expr.contains('text')}` → result != null && result.contains("text")
- `${expr.size > 0}` → result != null && !result.isEmpty()

snap-agent:
  cost:
    enabled: false                  # 成本核算总开关
    pricing:
      input: 3.00                   # 每 1M input tokens (CNY)

      cache-read: 0.30              # 每 1M cache-read tokens

      per-user-daily: 10.00         # 每用户每天 (可空=不限制)
