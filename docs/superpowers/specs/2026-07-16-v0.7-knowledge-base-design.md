## 1. 目标

让 Agent 具备业务领域知识，从"查数据"升级为"懂业务"，回答不再脱离上下文。

本次交付范围：
- `KnowledgeBase` SPI：知识源管理 + 关键词检索 `search(query, topK)`
- `KnowledgeSource` SPI：知识源加载接口（Markdown / 文件 / 外部 API）
- `MarkdownKnowledgeSource`：从 Markdown 文件加载知识，按 `##` 标题自动分段
- `SimpleKeywordSearcher`：基于词频重叠的简单检索（TF-IDF-lite，无向量嵌入）
- `KnowledgeInjector`：实现 `SystemPromptExtender`（v0.3 已有 SPI），运行时按用户问题检索知识片段注入 system prompt
- 配置：`snap-agent.knowledge.*`
- 内置知识示例：`docs/knowledge/` 目录

不在本次范围（延后到 v0.7.1+）：
- `ConversationKnowledgeSource`（历史对话提取 Q&A 对，需摘要能力）— v0.7.1
- `ExternalApiKnowledgeSource`（Confluence/语雀对接）— v0.7.1
- 向量嵌入语义检索（需 embedding model）— v0.7.2
- 知识沉淀闭环（对话标注 + 自动摘要入库 + 热重载 WatchService）— v0.7.1

### 数据流

```
启动时:
  SnapAgentAutoConfiguration
    → 收集所有 KnowledgeSource bean (MarkdownKnowledgeSource + 自定义)
    → 创建 KnowledgeBase (注入 sources + searcher)
    → KnowledgeInjector 装配为 SystemPromptExtender

运行时:
  AgentExecutor.execute()
    → buildSystemPrompt()
      → KnowledgeInjector.extend(skill, task)
        → 从 task.inputs 提取用户问题
        → knowledgeBase.search(question, maxFragments)
          → 遍历 sources 的 fragments
          → searcher.score(question, fragment) for each
          → 排序取 top-K (score >= minScore)
        → 格式化知识片段为 system prompt section
        → 返回 (或空字符串 if 无匹配)
    → system prompt 含业务知识 → LLM 回答更精准
```

### 与 v0.3 SystemPromptExtender 的关系
- v0.3 的 `SystemPromptExtender.extend(SkillMeta, AgentTask) → String` SPI 已存在
- v0.3 的 `ProjectContextExtender` 注入项目结构摘要
- v0.7 的 `KnowledgeInjector` 是第二个 `SystemPromptExtender` 实现
- **问题**: `AgentExecutor` 当前只接受单个 extender。需改为支持多个 extender
- **方案**: 将 `SystemPromptExtender` 扩展为 `List<SystemPromptExtender>`，按顺序调用，拼接结果

### 3.4 KnowledgeBase

```java
package cn.watsontech.snapagent.core.knowledge;

import java.util.ArrayList;
import java.util.List;

/**
 * 知识库, 管理 sources + 提供检索接口。
 */
public class KnowledgeBase {

    private final List<KnowledgeSource> sources;
    private final KnowledgeSearcher searcher;
    private volatile List<KnowledgeFragment> allFragments; // 启动时加载, 缓存

    public KnowledgeBase(List<KnowledgeSource> sources, KnowledgeSearcher searcher) {
        this.sources = sources;
        this.searcher = searcher;
        this.allFragments = loadAll();
    }

    /** 检索: 返回与 query 最相关的 topK 个片段 (score >= minScore)。 */
    public List<KnowledgeFragment> search(String query, int topK, double minScore) {
        if (query == null || query.isEmpty() || allFragments.isEmpty()) {
            return new ArrayList<>();
        }
        List<ScoredFragment> scored = new ArrayList<>();
        for (KnowledgeFragment f : allFragments) {
            double s = searcher.score(query, f);
            if (s >= minScore) {
                scored.add(new ScoredFragment(f, s));
            }
        }
        scored.sort((a, b) -> Double.compare(b.score, a.score));
        List<KnowledgeFragment> result = new ArrayList<>();
        for (int i = 0; i < Math.min(topK, scored.size()); i++) {
            result.add(scored.get(i).fragment);
        }
        return result;
    }

    /** 重新加载所有源 (热重载)。 */
    public void reload() {
        this.allFragments = loadAll();
    }

    public int size() { return allFragments.size(); }

    private List<KnowledgeFragment> loadAll() {
        List<KnowledgeFragment> all = new ArrayList<>();
        for (KnowledgeSource source : sources) {
            try {
                List<KnowledgeFragment> fragments = source.load();
                if (fragments != null) all.addAll(fragments);
            } catch (Exception e) {
                log.warn("Knowledge source {} load failed: {}", source.type(), e.getMessage());
            }
        }
        return all;
    }

    private static class ScoredFragment {
        final KnowledgeFragment fragment;
        final double score;
        ScoredFragment(KnowledgeFragment f, double s) { fragment = f; score = s; }
    }
}
```

## 4. Starter 层实现

### 4.1 MarkdownKnowledgeSource

从 Markdown 文件目录加载知识，按 `## ` 标题自动分段。

```java
package cn.watsontech.snapagent.boot2x.knowledge;

public class MarkdownKnowledgeSource implements KnowledgeSource {
    private final String dir;          // 目录路径 (classpath: 或文件系统)
    private final String sourceType;  // "markdown"

    @Override
    public List<KnowledgeFragment> load() {
        // 1. 解析目录路径 (classpath: → ClassPathResource; 其他 → 文件系统)
        // 2. 递归扫描 .md 文件
        // 3. 对每个文件:
        //    - 按 ## 分割 (正则: ^##\\s)
        //    - 每段: title = 标题行文本, content = 该段正文, source = "filename:section-N"
        //    - 文件级标题 (# 标题) 作为 metadata 的 "category"
        // 4. 返回所有片段
    }

    @Override
    public void reload() { /* load() 重新调用即可 */ }

    @Override
    public String type() { return "markdown"; }
}
```

**分段规则:**
- 文件以 `# ` 开头的行 = 文件标题 → 存为 metadata.category
- 文件以 `## ` 开头的行 = 章节标题 → 每个章节是一个 fragment
- `## ` 之前的内容（文件标题 + 简介）→ 一个 "overview" fragment
- 无 `## ` 的文件 → 整个文件作为一个 fragment

### 4.2 SimpleKeywordSearcher

基于词频重叠的简单评分。

```java
package cn.watsontech.snapagent.boot2x.knowledge;

public class SimpleKeywordSearcher implements KnowledgeSearcher {

    @Override
    public double score(String query, KnowledgeFragment fragment) {
        // 1. 分词: query 按空格/标点分词, 转小写, 去停用词(简单: 去 <2字符的词)
        // 2. 对中文: 按字符 bigram 分词 (无结巴分词依赖)
        // 3. fragment 分词: title + content
        // 4. 计算: query 词在 fragment 中出现的比例
        //    score = matchedWords / totalQueryWords
        // 5. title 匹配加权: title 中命中的词权重 ×2
        // 6. 返回 [0.0, 1.0]
    }
}
```

**评分算法:**
- 英文: 按空格分词, 转小写
- 中文: 按字符 bigram (2-gram) 分词, 如"补货策略" → ["补货", "货策", "策略"]
- score = (title 命中数 × 2 + content 命中数) / (query 词数 × 2)
- 截断到 [0.0, 1.0]

### 4.3 KnowledgeInjector

实现 `SystemPromptExtender`，运行时检索知识注入 system prompt。

```java
package cn.watsontech.snapagent.boot2x.knowledge;

public class KnowledgeInjector implements SystemPromptExtender {

    private final KnowledgeBase knowledgeBase;
    private final int maxFragments;
    private final double minScore;

    @Override
    public String extend(SkillMeta skill, AgentTask task) {
        // 1. 从 task inputs 提取查询文本
        //    拼接所有 input values 作为 query
        String query = buildQuery(task);
        if (query == null || query.isEmpty()) return "";

        // 2. 检索
        List<KnowledgeFragment> fragments =
            knowledgeBase.search(query, maxFragments, minScore);
        if (fragments.isEmpty()) return "";

        // 3. 格式化
        StringBuilder sb = new StringBuilder();
        sb.append("\n## 业务知识参考 (来自知识库)\n");
        sb.append("以下是与当前问题相关的业务知识片段:\n\n");
        for (int i = 0; i < fragments.size(); i++) {
            KnowledgeFragment f = fragments.get(i);
            sb.append("### 知识片段 ").append(i + 1).append(": ")
              .append(f.getTitle()).append("\n");
            sb.append("> 来源: ").append(f.getSource()).append("\n\n");
            sb.append(f.getContent()).append("\n\n");
        }
        sb.append("请结合以上业务知识回答用户问题。\n");
        return sb.toString();
    }

    private String buildQuery(AgentTask task) {
        if (task == null) return null;
        Map<String, String> inputs = task.getInputs();
        if (inputs == null || inputs.isEmpty()) return null;
        return String.join(" ", inputs.values());
    }
}
```

### 4.4 AgentExecutor 多 extender 支持

**修改 `AgentExecutor`**:
- 将 `SystemPromptExtender promptExtender`（单个）改为 `List<SystemPromptExtender> promptExtenders`（列表）
- `buildSystemPrompt()` 遍历列表, 依次调用 `extend()`, 拼接结果
- 向后兼容: 旧的 6 参构造器（接受单个 extender）改为接受 `List` 或用 `ObjectProvider<SystemPromptExtender>` 收集所有 bean

**实际实现**:
- 修改 `SnapAgentAutoConfiguration.agentExecutor()` bean:
  - 用 `ObjectProvider<SystemPromptExtender> extenderProvider` 收集所有 extender bean
  - 传给 AgentExecutor: `extenderProvider.orderedStream().collect(toList())`
- AgentExecutor 构造器改为接受 `List<SystemPromptExtender>`
- 旧的单 extender 构造器委托为 `Collections.singletonList(extender)`

```markdown
# SnapAgent 业务知识示例

## 系统概述
SnapAgent 是嵌入式 LLM 诊断 Agent，提供只读数据库查询、日志分析、代码理解、运营诊断能力。

## 数据库诊断
数据库诊断基于独立只读数据源连接。Agent 可查询业务数据但不能修改。
常见问题: 连接池打满 → 检查 max-pool-size 配置; 慢查询 → 用 EXPLAIN 分析执行计划。

适用于"这段逻辑为什么这样写"和"这个接口在哪实现"类问题。

## 运营诊断
运营诊断对接 Prometheus/Loki/Jaeger，可查询指标、搜索日志、分析调用链。
适用于"线上接口为什么慢"和"错误率为什么升高"类问题。
```
