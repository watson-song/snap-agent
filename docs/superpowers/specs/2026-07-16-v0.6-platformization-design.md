## 1. 目标

从框架升级为平台，支持更多环境配置和更细粒度的权限控制。

本次交付范围：
- 多环境数据源：`jdbc.datasources` 配置多 DSN，Skill `inputs.env` 选环境
- Skill 级别权限控制：每个 Skill 可声明 `required-permission`，不同角色可用不同 Skill
- REST API SDK：Java 客户端类 + REST API 文档
- 多 LLM 适配文档：通义/文心/智谱（OpenAI 兼容）配置指南

不在本次范围（延后到 v0.6.1+）：
- Spring Boot 3.x starter（jakarta.servlet，需 Java 17 独立模块）— v0.6.1

- 多租户 Agent 对话隔离 — v0.6.3
- 国产 LLM 原生 SDK 适配（非 OpenAI 兼容的）— v0.6.1

## 2. 多环境数据源

### 2.1 现状
`JdbcQueryToolProvider` 接受单个 `DataSource` bean，通过 `jdbc.datasource-bean-name` 配置。不支持多环境切换。

### 2.2 设计

**配置扩展:**
```yaml
snap-agent:
  jdbc:
    enabled: true
    datasource-bean-name: snapAgentReadOnlyDataSource   # 旧: 单 DSN (向后兼容)
    datasources:                                         # 新: 多环境 DSN
      sit:
        url: "jdbc:mysql://sit-db:3306/orders"
        username: "readonly_user"
        password: "${SIT_DB_PASSWORD:}"
        driver-class-name: "com.mysql.cj.jdbc.Driver"
      uat:
        url: "jdbc:mysql://uat-db:3306/orders"
        username: "readonly_user"
        password: "${UAT_DB_PASSWORD:}"
        driver-class-name: "com.mysql.cj.jdbc.Driver"
    default-env: ""    # 默认环境名, 空=第一个 datasources 条目
```

**DataSourceRegistry** (新类):
- 管理 `Map<String, DataSource>` — 按环境名查找
- 如果配置了 `datasources` → 为每个环境创建 `SimpleDriverDataSource`（Spring JDBC 自带, 无连接池开销, 只读诊断够用）
- 如果只配置了 `datasource-bean-name` → 单环境, key="default"
- `DataSource resolve(String env)` — env 为空返回 default, 不存在抛异常

**JdbcQueryToolProvider 扩展:**
- 构造器新增 `DataSourceRegistry` 可选参数
- Schema 新增 `env` 参数: `{"type":"string","description":"环境名 (e.g. sit/uat). 空则用默认环境."}`
- `execute()`: 从 args 提取 `env`, 调用 `registry.resolve(env)` 获取 DataSource
- 向后兼容: 如果 DataSourceRegistry 为 null, 用原来的单个 DataSource

## 3. Skill 级别权限控制

### 3.1 现状
权限控制仅在 controller 层: `securityGateway.hasPermission(requiredPermission)` 检查全局 `snap-agent.security.required-permission`。所有 Skill 共用同一权限码, 无法做细粒度控制。

### 3.2 设计

**SkillMeta 扩展:**
新增 `requiredPermission` 字段 (String, 默认空):
```java
private String requiredPermission = "";
```

**Skill frontmatter:**
```yaml
---
name: database-query
description: "..."
tools: [mysql_query]
required-permission: snap-agent:db-query   # 新增: 运行此 skill 需要的权限码
---
```

**SkillLoader 解析:**
解析 frontmatter 中的 `required-permission` 字段, 填入 SkillMeta。

**Controller 运行时检查:**
`POST /runs` 时, 从 SkillRegistry 获取 skill, 检查:
```java
if (!skill.getRequiredPermission().isEmpty()
    && !securityGateway.hasPermission(skill.getRequiredPermission())) {
    return 403; // "You don't have permission to run skill: X"
}
```

**GET /skills 响应:**
每个 skill DTO 新增 `requiredPermission` 字段。

### 3.3 安全语义
- `required-permission` 为空 → 继承全局 `security.required-permission` (向后兼容)
- `required-permission` 非空 → 覆盖全局权限, 仅检查 skill 级别
- 宿主可配置: 全局 `required-permission: snap-agent:access` + skill 级 `snap-agent:db-query` → 用户需同时有两者 (全局 access + 具体 skill 权限)

## 4. REST API SDK

### 4.1 设计

新建模块 `snap-agent-client` — 轻量 Java 客户端, 无 Spring 依赖:

```
snap-agent-client/
├── pom.xml                          (依赖 snap-agent-core, jackson-databind)
└── src/main/java/cn/watsontech/snapagent/client/
    ├── SnapAgentClient.java          ← 主客户端类
    ├── SnapAgentClientException.java ← 异常
    └── dto/
        ├── SkillDto.java             ← Skill 信息
        ├── RunRequest.java           ← 运行请求
        ├── RunStatus.java            ← 运行状态枚举
        └── TranscriptEvent.java       ← 事件
```

**SnapAgentClient API:**
```java
public class SnapAgentClient {
    private final String baseUrl;   // http://host:port/snap-agent
    private final String auth;      // Basic auth header value

    public SnapAgentClient(String baseUrl);
    public SnapAgentClient(String baseUrl, String username, String password);

    public List<SkillDto> listSkills();
    public SkillDto getSkill(String name);
    public String runSkill(String skillName, Map<String,String> inputs); // returns taskId
    public String getRunStatus(String taskId);
    public List<Map<String,Object>> getTranscript(String taskId);
    public void cancelRun(String taskId);
    public Map<String,Object> getUserInfo();
}
```

使用 HttpURLConnection (与 ObservabilityHttpClient 同模式), 不引入额外依赖。

`docs/embeed-skills-agent/10-rest-api.md` — 完整 REST API 参考。
