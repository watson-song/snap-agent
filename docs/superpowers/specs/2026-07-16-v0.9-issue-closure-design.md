## 1. 目标

从"一次性诊断"升级为"问题→诊断→方案→修复→验证→沉淀"的完整闭环。

当前 Agent 诊断完就结束，用户拿到根因后还要自己想方案、追踪修复、验证、沉淀经验。v0.9 补齐这四环。

本次交付范围：
- `IssueClosure` 数据模型：诊断→方案→修复→验证→沉淀 全生命周期记录
- `IssueStore` SPI：问题闭环记录持久化
- `IssueTracker` SPI：外部 Issue 系统（Jira/工单）对接
- `FileIssueStore`：JSON 文件存储默认实现
- `NoopIssueTracker`：默认空实现
- `IssueClosureService`：编排层 — 从诊断任务提取根因→生成方案→创建 Issue→验证→关闭+沉淀
- Controller 端点：`POST /runs/{id}/solution` / `POST /runs/{id}/issue` / `GET /issues/{id}` / `POST /issues/{id}/verify` / `POST /issues/{id}/close`
- `KnowledgeSedimentationExtractor`：从对话记录提取"问题→根因→方案"三元组，存入 v0.7 KnowledgeBase
- 配置：`snap-agent.issue-closure.*`
- 内置 Skill：`solution-suggest.md` / `verify-fix.md`

不在本次范围（延后到 v0.9.1+）：
- Jira/GitHub IssueTracker 实现（需各平台 API 适配）— v0.9.1
- 自动 PR 创建（需 Git 平台 API）— v0.9.1
- 自动验证回归（定时重跑验证）— v0.9.2

### 数据流

```
诊断完成:
  POST /runs/{taskId}/solution
    → IssueClosureService.proposeSolution(taskId)
      → 从 TaskStore 加载诊断任务 → 提取 rootCause (task.getReport())
      → 构造 AgentTask(skill="solution-suggest", inputs={rootCause, taskId})
      → agentExecutor.execute(task) → LLM 生成方案建议
      → 创建/更新 IssueClosure (status=SOLUTION_PROPOSED)
      → 返回方案列表

创建 Issue:
  POST /runs/{taskId}/issue
    → IssueClosureService.createIssue(taskId, selectedSolution)
      → issueTracker.createIssue(title, description) → issueId
      → 更新 IssueClosure (issueId, status=FIX_IN_PROGRESS)
      → 返回 issueId

验证:
  POST /issues/{issueId}/verify
    → IssueClosureService.verify(issueId)
      → 构造 AgentTask(skill="verify-fix", inputs={rootCause, originalQuery})
      → agentExecutor.execute(task) → LLM 验证
      → 更新 IssueClosure (status=VERIFIED, verificationResult)

关闭+沉淀:
  POST /issues/{issueId}/close
    → IssueClosureService.close(issueId)
      → KnowledgeSedimentationExtractor.extract(issueClosure) → KnowledgeFragment
      → knowledgeBase 缓存 + 可选持久化到 Markdown 文件
      → 更新 IssueClosure (status=CLOSED, knowledgeEntryId)
```

### 与 v0.5/v0.7 的关系
- v0.5 `AnomalyEventListener` 构造 AgentTask 交给 AgentExecutor 执行 — v0.9 `IssueClosureService` 复用此模式
- v0.7 `KnowledgeBase` 存储知识片段 — v0.9 `KnowledgeSedimentationExtractor` 产出的三元组存入此处
- v0.7 `SystemPromptExtender` 多 extender — v0.9 沉淀的知识自然被 `KnowledgeInjector` 注入后续对话

### 4.4 KnowledgeSedimentationExtractor

```java
package cn.watsontech.snapagent.boot2x.issue;

/** 从 IssueClosure 提取"问题→根因→方案"知识片段。 */
public class KnowledgeSedimentationExtractor {

    /** 提取知识片段。 */
    public KnowledgeFragment extract(IssueClosure issue) {
        String title = "问题: " + truncate(issue.getUserQuery(), 60);
        StringBuilder content = new StringBuilder();
        content.append("## 问题\n").append(issue.getUserQuery()).append("\n\n");
        content.append("## 根因\n").append(issue.getRootCause()).append("\n\n");
        content.append("## 解决方案\n");
        if (issue.getSelectedSolution() != null) {
            content.append(issue.getSelectedSolution()).append("\n");
        } else if (issue.getSolutions() != null) {
            for (String s : issue.getSolutions()) {
                content.append("- ").append(s).append("\n");
            }
        }
        if (issue.getVerificationResult() != null) {
            content.append("\n## 验证结果\n").append(issue.getVerificationResult()).append("\n");
        }
        return new KnowledgeFragment(
            title,
            content.toString(),
            "sedimentation:" + issue.getIssueId(),
            Collections.singletonMap("category", "经验沉淀")
        );
    }
}
```

## 6. 内置 Skill

### 6.1 solution-suggest.md

```markdown
---
name: solution-suggest
description: 基于诊断根因生成解决方案建议
inputs:
  - name: root_cause
    description: 根因摘要
    required: true
  - name: original_query
    description: 用户原始问题
    required: true
  - name: task_id
    description: 关联诊断任务 ID
    required: false
---

# 方案建议生成

你是一个运维诊断专家。根据根因分析结果，给出 2-3 个候选解决方案。

## 步骤

1. 理解根因: `{root_cause}`
2. 结合用户原始问题: `{original_query}`
3. 如需, 使用 code_graph_impact_analysis 查看修改影响范围
4. 生成方案:
   - 方案1: [描述] (推荐程度: 高/中/低)
   - 方案2: [描述] (推荐程度: 高/中/低)
5. 推荐一个方案并说明理由

## 输出格式

方案1: [描述]
推荐程度: 高
理由: [为什么推荐]

方案2: [描述]
推荐程度: 中
理由: [为什么]

推荐: 方案1 (临时) + 方案2 (长期)
相关代码: [文件:行号]
```
