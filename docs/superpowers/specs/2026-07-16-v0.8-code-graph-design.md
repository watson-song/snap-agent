## 1. 目标

从"按文件名读代码"升级为"按语义关系导航代码"，Agent 理解调用链、依赖关系、业务领域边界。

本次交付范围：
- `CodeGraph` 核心数据模型：节点（类/方法/字段）+ 边（调用/继承/实现/依赖）
- `CodeGraphBuilder` SPI：从源码构建代码图谱
- `CodeGraphIndex` SPI：图谱存储与检索
- `SimpleCodeGraphBuilder`：基于正则的轻量 Java 源码解析（无 JavaParser 依赖）
- `InMemoryCodeGraphIndex`：内存图谱存储
- `CodeGraphToolProvider`：Agent 工具（调用链/反向调用/影响分析/模式查找）
- 配置：`snap-agent.code-graph.*`

不在本次范围（延后到 v0.8.1+）：
- JavaParser/Spoon 完整 AST 解析（需额外依赖）— v0.8.1
- SQLite/H2 持久化索引 — v0.8.1
- WatchService 源码变更监听 + 增量更新 — v0.8.1
- 语义搜索（需 embedding model）— v0.8.2

## 2. 架构总览

```
snap-agent-core (SPI 层)
├── codegraph/
│   ├── CodeGraph.java              ← 数据模型: 节点 + 边集合
│   ├── CodeGraphNode.java          ← 节点: ClassNode / MethodNode / FieldNode
│   ├── CodeGraphEdge.java          ← 边: CALLS / IMPLEMENTS / EXTENDS / DEPENDS_ON
│   ├── CodeGraphBuilder.java       ← SPI: 从源码构建图谱
│   └── CodeGraphIndex.java         ← SPI: 存储与检索

snap-agent-spring-boot-2x-starter (实现层)
├── codegraph/
│   ├── SimpleCodeGraphBuilder.java ← 正则解析 Java 源码, 提取类/方法/字段/调用
│   ├── InMemoryCodeGraphIndex.java← 内存图谱存储
│   └── CodeGraphToolProvider.java  ← Agent 工具: 调用链/反向链/影响分析
├── autoconfig/
│   ├── SnapAgentProperties.java    ← 新增 CodeGraph 嵌套类
│   └── SnapAgentAutoConfiguration.java ← 新增 code-graph bean 装配
```

### 数据流

```
启动时:
  SnapAgentAutoConfiguration
    → 创建 SimpleCodeGraphBuilder (注入 CodePathGuard + scanPackages)
    → builder.build() → CodeGraph (节点+边)
    → 创建 InMemoryCodeGraphIndex (注入 CodeGraph)
    → CodeGraphToolProvider 装配为 ToolProvider

运行时:
  LLM 调用 code_graph_call_chain(methodSignature)
    → CodeGraphToolProvider.execute()
      → index.findCallChain(methodName)
        → 从 MethodNode 出发, 沿 CALLS 边 BFS 遍历
      → 格式化为文本返回给 LLM
```

### 与 v0.3/v0.7 的关系
- v0.3 `CodeReaderToolProvider`：读单文件内容（"这个文件里写了什么"）
- v0.8 `CodeGraphToolProvider`：查询代码间关系（"谁调用了谁、改了影响什么"）
- v0.7 `KnowledgeBase`：业务知识注入
- 三者互补：知识库提供业务上下文，图谱提供代码关系，Reader 提供源码细节

## 3. Core 层数据模型

### 3.1 CodeGraphNode

```java
package cn.watsontech.snapagent.core.codegraph;

/** 代码图谱节点。一个节点代表一个类、方法或字段。 */
public class CodeGraphNode {
    public enum NodeType { CLASS, METHOD, FIELD }

    private final String id;        // 唯一标识: "com.example.Foo" 或 "com.example.Foo#bar(String)"
    private final NodeType type;
    private final String name;      // 简单名: "Foo" / "bar"
    private final String packageName; // "com.example"
    private final String className;  // 所属类 (METHOD/FIELD 用)
    private final String returnType; // 方法返回类型 / 字段类型 (CLASS 为空)
    private final String filePath;   // 源文件路径
    private final int lineNumber;    // 行号

    // 构造器 + getter + toString
}
```

### 3.2 CodeGraphEdge

```java
package cn.watsontech.snapagent.core.codegraph;

/** 代码图谱边。一条边代表两个节点间的语义关系。 */
public class CodeGraphEdge {
    public enum EdgeType { CALLS, IMPLEMENTS, EXTENDS, DEPENDS_ON, OVERRIDES, REFERENCES }

    private final String fromId;    // 源节点 ID
    private final String toId;      // 目标节点 ID
    private final EdgeType type;
    private final String context;   // 调用上下文 (e.g. "第45行")

    // 构造器 + getter + toString
}
```

### 3.3 CodeGraph

```java
package cn.watsontech.snapagent.core.codegraph;

import java.util.*;

/** 不可变代码图谱。包含节点集合和边集合。 */
public class CodeGraph {
    private final List<CodeGraphNode> nodes;
    private final List<CodeGraphEdge> edges;

    public CodeGraph(List<CodeGraphNode> nodes, List<CodeGraphEdge> edges) {
        this.nodes = new ArrayList<>(nodes);
        this.edges = new ArrayList<>(edges);
    }

    public List<CodeGraphNode> getNodes() { return Collections.unmodifiableList(nodes); }
    public List<CodeGraphEdge> getEdges() { return Collections.unmodifiableList(edges); }
    public int nodeCount() { return nodes.size(); }
    public int edgeCount() { return edges.size(); }
}
```

### 3.5 CodeGraphIndex (SPI)

```java
package cn.watsontech.snapagent.core.codegraph;

import java.util.*;

/** 代码图谱索引 SPI。提供检索接口。 */
public interface CodeGraphIndex {
    /** 按名称查找节点 (模糊匹配)。 */
    List<CodeGraphNode> findByName(String namePattern);

    /** 获取节点的所有出边 (此节点指向哪些节点)。 */
    List<CodeGraphEdge> getOutgoingEdges(String nodeId);

    /** 获取节点的所有入边 (哪些节点指向此节点)。 */
    List<CodeGraphEdge> getIncomingEdges(String nodeId);

    /** 正向调用链: 从指定方法出发, 沿 CALLS 边 BFS 遍历。 */
    List<CodeGraphNode> findCallChain(String methodId, int maxDepth);

    /** 反向调用链: 谁调用了指定方法。 */
    List<CodeGraphNode> findReverseCallChain(String methodId, int maxDepth);

    /** 影响分析: 修改指定类/方法会影响哪些下游。 */
    List<CodeGraphNode> findImpactScope(String nodeId, int maxDepth);

    /** 根据 ID 获取节点。 */
    CodeGraphNode getNode(String id);
}
```

## 4. Starter 层实现

### 4.1 SimpleCodeGraphBuilder

基于正则表达式解析 Java 源码。无需 JavaParser 依赖，轻量但精度有限。

**解析规则:**
- 包声明: `package\s+([\w.]+)\s*;`
- 类声明: `(public\s+)?(abstract\s+)?(class|interface|enum)\s+(\w+)(\s+extends\s+([\w.]+))?(\s+implements\s+([\w.,\s]+))?`
- 方法声明: `(public|protected|private)\s+(?:static\s+)?([\w.<>\[\]]+)\s+(\w+)\s*\(([^)]*)\)`
- 方法调用: `\b(\w+)\.(\w+)\s*\(` (实例方法调用) 或直接 `(\w+)\s*\(` (本类方法调用)
- 字段声明: `(private|protected|public)\s+(?:static\s+)?(?:final\s+)?([\w.<>\[\]]+)\s+(\w+)\s*[;=]`

**局限（诚实声明）:**
- 正则无法处理注释内的假阳性（注释中提到的方法调用会被误提取）
- 无法区分重载方法（同名的不同参数列表）
- 无法解析 lambda/Stream 内的方法调用
- 泛型类型擦除（`List<String>` 的字段类型会保留泛型）
- 这些限制在文档中明确说明，v0.8.1 用 JavaParser 替换后可解决

### 4.2 InMemoryCodeGraphIndex

使用 `Map<String, CodeGraphNode>` + `Map<String, List<CodeGraphEdge>>` 双向邻接表存储。

- `findByName`: 遍历节点列表, `name.contains(pattern)` 或 `id.contains(pattern)` 模糊匹配
- `findCallChain`: BFS 从方法节点出发, 沿 `CALLS` 类型的出边遍历, 返回层级路径
- `findReverseCallChain`: BFS 从方法节点出发, 沿 `CALLS` 类型的入边遍历
- `findImpactScope`: 从节点出发, 沿入边遍历 (谁依赖了我 → 谁依赖了依赖我的人), 返回受影响节点列表

### 4.3 CodeGraphToolProvider

提供 4 个工具:

| 工具 | 输入 schema | 输出 |
|------|-----------|------|
| `code_graph_call_chain` | `methodSignature` (String), `maxDepth` (Int, default 5) | 正向调用链文本 |
| `code_graph_reverse_chain` | `methodSignature` (String), `maxDepth` (Int, default 5) | 反向调用链文本 |
| `code_graph_impact_analysis` | `classNameOrMethod` (String), `maxDepth` (Int, default 3) | 影响范围列表 |
| `code_graph_find` | `pattern` (String) | 匹配的类/方法列表 |

工具 schema 以 JSON 返回，被 `ToolDispatcher` 自动发现。

## 5. 配置树

```yaml
snap-agent:
  code-graph:
    enabled: false               # 代码图谱总开关
    scan-packages:               # 扫描的包路径
      - com.example.order
      - com.example.inventory
    max-depth: 5                 # 调用链最大深度
    max-impact-depth: 3          # 影响分析最大深度
```

### 装配条件

| Bean | 装配条件 |
|------|---------|
| `SimpleCodeGraphBuilder` | `code-graph.enabled=true` 且 `code.enabled=true` (复用 CodePathGuard) |
| `InMemoryCodeGraphIndex` | `code-graph.enabled=true` |
| `CodeGraphToolProvider` | `code-graph.enabled=true` (作为 ToolProvider) |
