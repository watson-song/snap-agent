## 1. 目标

Agent 不止能查数据，还能读懂宿主项目代码，回答更精准。

本次交付范围：
- 三个只读工具：`code_read`、`project_structure`、`git_log`
- 项目上下文注入：启动时扫描项目结构，生成摘要注入 system prompt

不在本次范围（延后到 v0.3.1 或 v0.4）：
- Skill 自动生成
- 代码引用渲染（前端可点击链接）

## 2. 架构总览

```
snap-agent-core (SPI 层)
├── agent/SystemPromptExtender.java       ← 新 SPI: 单方法接口
└── agent/AgentExecutor.java              ← 扩展: 构造函数加可选 SystemPromptExtender

snap-agent-spring-boot-2x-starter (实现层)
├── tool/
│   ├── CodePathGuard.java                ← 路径安全守卫 (复用 LogPathGuard 模式)
│   ├── CodeReaderToolProvider.java       ← code_read 工具
│   ├── ProjectStructureToolProvider.java ← project_structure 工具
│   └── GitLogToolProvider.java           ← git_log 工具
├── context/
│   └── ProjectContextExtender.java       ← 实现 SystemPromptExtender, 启动时扫描缓存
└── autoconfig/
    └── SnapAgentAutoConfiguration.java   ← 新增 bean 装配
```

### 数据流

```
启动时:
  SnapAgentAutoConfiguration
    → CodePathGuard (从 props 构造, 校验 project-root)
    → ProjectContextExtender (扫描项目结构, 生成摘要缓存)
    → 3 个 ToolProvider bean (各自注入 CodePathGuard)

运行时:
  AgentExecutor.execute()
    → buildSystemPrompt()
      → 如果 SystemPromptExtender != null, 追加项目上下文摘要
    → LLM 多轮循环
      → LLM 调用 code_read / project_structure / git_log
      → ToolDispatcher 分发到对应 ToolProvider
      → CodePathGuard.validate() 校验路径安全
```

### 关键约束

- `snap-agent.code.enabled=false`（默认），不启用时不创建任何 bean
- `snap-agent.code.project-root` 为空时，即使 enabled=true 也不创建工具 bean（仅 WARN 日志）
- 所有三个工具共享同一个 `CodePathGuard` 实例

## 3. SystemPromptExtender SPI (core)

```java
package cn.watsontech.snapagent.core.agent;

/**
 * System prompt 扩展点。在 AgentExecutor 组装 system prompt 时调用，
 * 允许实现方向提示中追加上下文信息（如项目结构摘要）。
 *
 * <p>这是 v0.7 KnowledgeInjector 的前向兼容基础：
 * v0.7 将扩展为按用户问题动态检索注入，v0.3 仅做启动时静态注入。</p>
 */
public interface SystemPromptExtender {

    /**
     * 追加到 system prompt 末尾的上下文内容。
     *
     * @param skill 当前执行的 Skill 元数据
     * @param task  当前任务
     * @return 上下文文本（可为空字符串，不可为 null）
     */
    String extend(SkillMeta skill, AgentTask task);
}
```

### AgentExecutor 变更

构造函数新增可选参数 `SystemPromptExtender`（可为 null）：

```java
public AgentExecutor(LlmClient llmClient, ToolDispatcher toolDispatcher,
                     TaskStore taskStore, int maxTurns, int maxTokens,
                     SystemPromptExtender promptExtender) {
    // ... 现有字段赋值 ...
    this.promptExtender = promptExtender;
}
```

旧五参数构造函数委托新构造函数（传 null），保持向后兼容：

```java
public AgentExecutor(LlmClient llmClient, ToolDispatcher toolDispatcher,
                     TaskStore taskStore, int maxTurns, int maxTokens) {
    this(llmClient, toolDispatcher, taskStore, maxTurns, maxTokens, null);
}
```

`buildSystemPrompt()` 末尾追加注入逻辑：

```java
String buildSystemPrompt(SkillMeta skill, AgentTask task) {
    StringBuilder sb = new StringBuilder();
    sb.append(READ_ONLY_PREFIX);
    // ... 现有逻辑不变 ...
    sb.append("userId: ").append(task.getUserId()).append("\n");

    // 项目上下文注入
    if (promptExtender != null) {
        String context = promptExtender.extend(skill, task);
        if (context != null && !context.isEmpty()) {
            sb.append("\n").append(context);
        }
    }

    return sb.toString();
}
```

## 4. CodePathGuard (starter)

复用 `LogPathGuard` 的设计模式，但面向源码文件读取场景。

```java
package cn.watsontech.snapagent.boot2x.tool;

public class CodePathGuard {

    private final Path projectRoot;           // 项目根目录（绝对、规范化）
    private final Set<String> allowedExtensions; // 允许的文件扩展名（小写，含点）
    private final int maxLines;               // 单次读取最大行数
    private final long maxFileBytes;           // 单个文件最大字节数

    public CodePathGuard(String projectRoot, List<String> allowedExtensions,
                         int maxLines, long maxFileBytes) { ... }

    /**
     * 校验文件路径是否安全可读。
     * @return Result（allowed + path，或 rejected + reason）
     */
    public Result validate(String filePath) { ... }

    /** 获取项目根目录 Path。 */
    public Path getProjectRoot() { ... }

    /** 不可变校验结果，与 LogPathGuard.Result 结构一致。 */
    public static final class Result { ... }
}
```

### 校验规则（按顺序）

1. **空路径** → 拒绝
2. **目录穿越** → 路径含 `..` → 拒绝
3. **规范化后必须在 projectRoot 之下** → `path.normalize().startsWith(projectRoot)` → 否则拒绝
4. **文件必须存在** → `Files.exists()` → 否则拒绝
5. **必须是普通文件** → `Files.isRegularFile()` → 否则拒绝
6. **扩展名必须在白名单内** → 小写化后匹配 → 否则拒绝
7. **文件大小不超过 maxFileBytes** → `Files.size()` → 否则拒绝

### 与 LogPathGuard 的差异

| 维度 | LogPathGuard | CodePathGuard |
|------|-------------|---------------|
| 根目录数量 | 多个 allowedRoots | 单个 projectRoot |
| 扩展名过滤 | 无（日志无扩展名约束） | 白名单 |
| 默认 maxFileBytes | 10MB | 512KB（源码文件通常较小） |
| 默认 maxLines | 500 | 500 |

## 5. CodeReaderToolProvider (starter)

```java
package cn.watsontech.snapagent.boot2x.tool;

public class CodeReaderToolProvider implements ToolProvider {

    @Override public String name() { return "code_read"; }

    @Override public String schema() {
        // JSON Schema: file_path (required), start_line (optional),
        //              end_line (optional), keyword (optional), max_lines (optional)
    }

    @Override
    public ToolResult execute(Map<String, Object> args, ToolContext ctx) {
        // 1. 提取 file_path
        // 2. CodePathGuard.validate(file_path)
        // 3. 提取 start_line, end_line, keyword, max_lines
        // 4. 读取文件行, 按 [start_line, end_line] 裁剪
        // 5. 如有 keyword, 过滤包含该关键词的行
        // 6. 截断到 max_lines
        // 7. 带行号格式化输出
    }
}
```


```
# Filter: keyword="createOrder"

  46│        // skipValidation 临时跳过校验
  47│        if (skipValidation) {
```

## 7. GitLogToolProvider (starter)

通过 `ProcessBuilder` 调用 `git` 命令，零 Java 依赖。

```java
package cn.watsontech.snapagent.boot2x.tool;

public class GitLogToolProvider implements ToolProvider {

    private final CodePathGuard pathGuard;

    @Override public String name() { return "git_log"; }

    @Override public String schema() {
        // JSON Schema: file_path (optional), mode (log|blame|show),
        //              max_entries (default 20), commit_hash (optional, show 模式)
    }

    @Override
    public ToolResult execute(Map<String, Object> args, ToolContext ctx) {
        // 1. 提取 mode, file_path, max_entries, commit_hash
        // 2. 如有 file_path, CodePathGuard.validate() 校验
        // 3. 构造 git 命令参数列表（不用 shell, 用 ProcessBuilder + 数组）
        // 4. 执行, 超时 10s, 捕获 stdout
        // 5. 截断到 max_entries
        // 6. 格式化输出
    }
}
```

```json
{
  "name": "git_log",
  "description": "查看项目的 git 历史。支持 log（提交历史）、blame（行级别作者）和 show（查看具体 commit）。",
  "input_schema": {
    "type": "object",
    "properties": {
      "file_path": {
        "type": "string",
        "description": "可选，限定到指定文件的 git 历史"
      },
      "mode": {
        "type": "string",
        "enum": ["log", "blame", "show"],
        "description": "操作模式：log=提交历史，blame=行级别作者，show=查看具体 commit",
        "default": "log"
      },
      "max_entries": {
        "type": "integer",
        "description": "最大返回条数（默认 20）",
        "default": 20
      },
      "commit_hash": {
        "type": "string",
        "description": "commit hash（仅 show 模式）"
      }
    }
  }
}
```

### 命令构造（命令注入防护）

所有参数通过 `ProcessBuilder(List<String>)` 传递，不走 shell：

| 模式 | 命令 |
|------|------|
| log | `git log --oneline -n {max} [-- {file_path}]` |
| blame | `git blame -l -n {max_lines} {file_path}` |
| show | `git show --stat {commit_hash}` |

**命令注入防护**：
- `file_path` 经过 `CodePathGuard.validate()` 后使用规范化路径
- `commit_hash` 正则校验 `^[0-9a-f]{7,40}$`
- `max_entries` 范围限制 1-100
- 不使用 `Runtime.exec(String)`，始终用 `ProcessBuilder(List<String>)`
- 工作目录设为 `projectRoot`

## 8. ProjectContextExtender (starter)

实现 `SystemPromptExtender`，启动时扫描项目结构并生成摘要，缓存结果。

```java

public class ProjectContextExtender implements SystemPromptExtender {

    private final String cachedSummary;  // 启动时生成一次

    public ProjectContextExtender(CodePathGuard pathGuard, int structureDepth) {
        this.cachedSummary = generateSummary(pathGuard.getProjectRoot(), structureDepth);
    }

    @Override
    public String extend(SkillMeta skill, AgentTask task) {
        return cachedSummary;
    }

    private String generateSummary(Path projectRoot, int depth) {
        // 1. 扫描目录树到 depth 层
        // 2. 识别模块（pom.xml 存在的目录）
        // 3. 统计每模块的 .java 文件数
        // 4. 生成 Markdown 摘要
    }
}
```

```markdown

模块:
- snap-agent-core (src/main/java/cn/watsontech/snapagent/core/) — 15 个 Java 文件
```

## 10. AutoConfiguration 装配

`SnapAgentAutoConfiguration` 新增 5 个 bean：

```java
// ---- CodePathGuard ----
// 使用 @ConditionalOnExpression 确保 project-root 非空时才创建 bean，
// 避免 null bean 导致 @ConditionalOnBean 失效
@Bean
@ConditionalOnExpression("'${snap-agent.code.enabled:false}' == 'true' and !'${snap-agent.code.project-root:}'.trim().isEmpty()")
@ConditionalOnMissingBean
public CodePathGuard codePathGuard(SnapAgentProperties props) {
    String root = props.getCode().getProjectRoot();
    Path rootPath = Paths.get(root).toAbsolutePath().normalize();
    if (!Files.isDirectory(rootPath)) {
        log.warn("CodePathGuard not assembled: project-root {} is not a directory", rootPath);
        return null;  // Spring Boot 2.x 不注册 null bean → 依赖它的 bean 也不会创建
    }
    log.info("CodePathGuard assembled with project-root: {}", rootPath);
    return new CodePathGuard(root, props.getCode().getAllowedExtensions(),
            props.getCode().getMaxLines(), props.getCode().getMaxFileBytes());
}

// ---- ProjectContextExtender ----
@Bean
@ConditionalOnProperty(prefix = "snap-agent.code", name = "enabled", havingValue = "true")

@ConditionalOnMissingBean
public SystemPromptExtender projectContextExtender(CodePathGuard codePathGuard,
                                                    SnapAgentProperties props) {
    return new ProjectContextExtender(codePathGuard, props.getCode().getStructureDepth());
}
```

## 11. 安全设计

### 路径安全（CodePathGuard）

- 目录穿越 `..` → 拒绝
- 路径规范化后必须在 projectRoot 之下
- 扩展名白名单防止读取敏感文件（如 `.env`、`.key`、`.pem`）
- 文件大小限制防止 OOM
- `project-root` 必须是实际存在的目录，启动时校验

### 命令注入防护（GitLogToolProvider）

- **不使用** `Runtime.exec(String)` — 始终用 `ProcessBuilder(List<String>)`
- `file_path` 经 CodePathGuard 规范化后使用
- `commit_hash` 正则校验 `^[0-9a-f]{7,40}$`
- `max_entries` 范围限制 1-100
- 进程超时 10 秒，超时自动 destroy
- 工作目录设为 projectRoot，不依赖 shell PATH

### 只读保证

- 三个工具均为只读：读文件、读目录、读 git 历史
- 不写入、不删除、不修改任何文件
- git 命令只用 `log`/`blame`/`show`，不用任何写操作

## 13. 内置 Skill

新增 `code-analysis.md` 内置 skill，放在 `docs/skills/` 下：

```markdown
---
name: code-analysis
description: 分析项目代码，回答"这段逻辑为什么这样写""这个接口在哪实现"等问题
inputs:
  - name: question
    description: 代码相关问题
---

## 分析步骤

1. **理解问题** — 确认用户想了解的类/方法/逻辑
2. **定位代码** — 调用 project_structure 扫描项目结构，定位目标文件
3. **读取源码** — 调用 code_read 读取相关文件内容
4. **追溯历史**（如需要）— 调用 git_log 查看变更历史
5. **输出结论** — 代码逻辑说明 + 变更历史（如有）+ 建议改进
```
